# ADR-005: Real-Time Push Strategy

## Status: Accepted

## Decision
WebSocket (ws library) via dedicated websocket-service

## Reasoning
- WebSocket supports full-duplex — dashboard can send actions too
- Dedicated service decouples push logic from business services
- Services POST events to websocket-service via HTTP
- Dashboard connects to single WS endpoint on port 3008
- SSE considered but WebSocket chosen for bidirectional capability
