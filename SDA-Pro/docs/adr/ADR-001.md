# ADR-001: SOA vs Microservices vs Modular Monolith

## Status: Accepted

## Decision
SOA (Service-Oriented Architecture)

## Reasoning
- Each service owns a bounded domain context
- Services communicate via REST (sync) and RabbitMQ (async)
- Simpler than full microservices for a 2-week project
- Avoids monolith coupling while keeping deployment manageable

## Consequences
- Each service has its own DB schema (shared PostgreSQL instance)
- Services can be scaled independently
- RabbitMQ decouples producers from consumers
