# ADR-002: Sync vs Async Communication

## Status: Accepted

## Decision
Hybrid approach:
- Synchronous (REST/gRPC): Dashboard → Services, Service-to-Service queries
- Asynchronous (RabbitMQ): All domain events (AlertIngested, IncidentCreated etc.)

## Reasoning
- REST for request-response patterns needing immediate results
- RabbitMQ for event-driven flows where decoupling matters
- Avoids tight coupling between ingestion and response pipeline
